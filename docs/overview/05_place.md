# ✂️ place — 블록 배치 & 분할

> 찾은 가용 블록에 실제로 메모리를 할당하는 함수.
> 남는 공간이 충분하면 블록을 **분할(split)** 하여 낭비를 최소화합니다.

---

## 코드

```c
static void place(void *bp, size_t asize)
{
    size_t csize = GET_SIZE(HDRP(bp));  // 가용 블록의 실제 크기

    if ((csize - asize) >= (2 * DSIZE)) {  // 남는 공간 >= 16B
        // 분할: 앞쪽 할당 + 뒤쪽 가용
        PUT(HDRP(bp), PACK(asize, 1));          // 할당 블록 헤더
        PUT(FTRP(bp), PACK(asize, 1));          // 할당 블록 풋터
        bp = NEXT_BLKP(bp);                     // bp → 나머지 공간
        PUT(HDRP(bp), PACK(csize - asize, 0));  // 가용 블록 헤더
        PUT(FTRP(bp), PACK(csize - asize, 0));  // 가용 블록 풋터
    } else {
        // 분할 불가: 통째로 할당
        PUT(HDRP(bp), PACK(csize, 1));
        PUT(FTRP(bp), PACK(csize, 1));
    }
}
```

---

## 분할 조건: 왜 2*DSIZE (16B)?

```
최소 블록 구조:
┌─────────┬──────────┬─────────┐
│ HDR(4B) │ 페이로드  │ FTR(4B) │
│         │ 최소 8B  │         │
└─────────┴──────────┴─────────┘
          총 = 16B

분할 후 남은 조각이 16B 미만이면?
→ 독립적인 블록으로 사용 불가
→ 그냥 통째로 할당하는 게 낫다 (internal fragmentation은 있지만 구조 유지)
```

---

## 분할하는 경우 — 시각화

```
csize = 64B, asize = 24B
남는 공간 = 64 - 24 = 40B ≥ 16B  → 분할!

[분할 전]
┌───────┬─────────────────────────────────────┬───────┐
│HDR    │              (가용 공간)              │  FTR  │
│ 64|0  │               56 bytes               │ 64|0  │
└───────┴─────────────────────────────────────┴───────┘
↑
bp

[분할 후]
┌───────┬────────────┬───────┬───────┬─────────────────┬───────┐
│HDR    │  PAYLOAD   │  FTR  │  HDR  │   (가용 공간)    │  FTR  │
│ 24|1  │  (16 bytes)│ 24|1  │ 40|0  │   (32 bytes)    │ 40|0  │
└───────┴────────────┴───────┴───────┴─────────────────┴───────┘
↑                            ↑
bp (할당 블록)              bp = NEXT_BLKP(bp) (가용 블록)
```

---

## 분할하지 않는 경우 — 시각화

```
csize = 24B, asize = 16B
남는 공간 = 24 - 16 = 8B < 16B  → 분할 불가!

[할당 전]
┌───────┬────────────┬───────┐
│HDR    │ (가용 공간) │  FTR  │
│ 24|0  │  (16 bytes)│ 24|0  │
└───────┴────────────┴───────┘

[할당 후: 통째로 24B 할당]
┌───────┬────────────┬───────┐
│HDR    │  PAYLOAD   │  FTR  │
│ 24|1  │  (16 bytes)│ 24|1  │
└───────┴────────────┴───────┘

※ 실제 필요 크기 16B인데 24B 할당됨 → 8B는 internal fragmentation
  하지만 어쩔 수 없음 (16B짜리 가용 블록은 사용 불가)
```

---

## 단편화(Fragmentation) 관점

| 상황 | 설명 |
|------|------|
| **Internal fragmentation** | 블록이 asize보다 클 때 남는 공간 (분할 불가 시) |
| **External fragmentation** | 총 가용 공간은 충분하지만 연속되지 않아 사용 못 할 때 |

분할을 통해 **external fragmentation을 줄이고**, 가용 블록을 최대한 작게 유지합니다.

---

## mm_malloc에서의 흐름

```
mm_malloc(size)
    ↓
asize 계산 (정렬 포함)
    ↓
find_fit(asize) → bp 반환
    ↓
place(bp, asize)
    ├─ 분할 가능 → 앞 asize 할당 + 뒤 나머지 가용
    └─ 분할 불가 → 전체 csize 할당
    ↓
return bp (사용자에게 payload 시작 주소 반환)
```
