# `ALIGN` — 8바이트 정렬 매크로

---

## 매크로 정의

```c
#define DSIZE 8
#define ALIGN(size) (((size) + (DSIZE-1)) & ~(DSIZE-1))

// DSIZE = 8이므로 실제로는:
#define ALIGN(size) (((size) + 7) & ~7)
```

---

## 역할 요약

`ALIGN(size)`는 `size`를 **8의 배수로 올림**하는 매크로입니다.
핵심 역할은 딱 두 가지입니다.

1. **크기 올림** — `+ (DSIZE-1)` : 다음 8의 배수 이상으로 넉넉하게 올림
2. **하위 비트 제거** — `& ~(DSIZE-1)` : 하위 3비트를 0으로 만들어 8의 배수로 확정

---

## 목표 — size를 8의 배수로 올림

```
ALIGN(1)  →  8
ALIGN(7)  →  8
ALIGN(8)  →  8   ← 이미 8의 배수면 그대로
ALIGN(9)  → 16
ALIGN(15) → 16
ALIGN(16) → 16
ALIGN(17) → 24
```

---

## 단계별 실행 과정

### Step 1 — `size + 7` : 여유분을 더한다

8의 배수로 올려야 하는데, 최대 7만큼 모자랄 수 있습니다.
그래서 먼저 `size + 7`을 해서 **무조건 다음 8의 배수 이상**으로 만들어 둡니다.

```
8의 배수: 0, 8, 16, 24, 32 ...

size=1  →  1+7 =  8   (딱 8의 배수에 걸림)
size=9  →  9+7 = 16   (딱 8의 배수에 걸림)
size=15 → 15+7 = 22   (8의 배수 16과 24 사이)
size=8  →  8+7 = 15   (8의 배수 8과 16 사이)
```

---

### Step 2 — `~7` : 하위 3비트를 지우는 마스크

`7`을 2진수로 쓰면 `0b...00000111`이고, `~7`은 비트를 뒤집은 `0b...11111000`입니다.

```
     7 = 0000 0000 0000 0000 0000 0000 0000 0111
    ~7 = 1111 1111 1111 1111 1111 1111 1111 1000
                                              ↑↑↑
                                         하위 3비트가 0
```

어떤 수에 `& ~7`을 하면 하위 3비트가 강제로 0이 됩니다.
하위 3비트가 0인 수 = **8의 배수**입니다.

```
8의 배수 확인:
 8 = 0b...0 1000  ← 하위 3비트 000
16 = 0b...1 0000  ← 하위 3비트 000
24 = 0b...1 1000  ← 하위 3비트 000
```

---

### Step 3 — `(size + 7) & ~7` : 합치면?

Step 1에서 넉넉하게 올린 값에서, Step 2로 하위 3비트를 잘라냅니다.

```
size = 9 예시:

  9 + 7 = 16 = 0b 0001 0000
  & ~7       = 0b 1111 1000
              ──────────────
               0b 0001 0000 = 16  ✓

size = 15 예시:

  15 + 7 = 22 = 0b 0001 0110
  & ~7        = 0b 1111 1000
               ──────────────
                0b 0001 0000 = 16  ✓

size = 8 예시 (이미 8의 배수):

  8 + 7 = 15 = 0b 0000 1111
  & ~7       = 0b 1111 1000
              ──────────────
               0b 0000 1000 = 8   ✓  그대로 유지
```

## 왜 `DSIZE - 1`을 쓰는가?

`DSIZE = 8`이고 `DSIZE - 1 = 7`입니다.
8로 정렬할 때 부족할 수 있는 최대 여유분이 `8 - 1 = 7`이기 때문입니다.

```
DSIZE = 16이면:
  ALIGN(size) = (size + 15) & ~15  → 16의 배수로 올림

DSIZE = 32이면:
  ALIGN(size) = (size + 31) & ~31  → 32의 배수로 올림
```

`DSIZE - 1`을 쓰면 `DSIZE` 값만 바꿔도 `ALIGN`이 자동으로 따라갑니다.

## 요약

```
ALIGN(size) = (size + 7) & ~7

  ├─ + 7    → 다음 8의 배수 이상으로 넉넉하게 올림
  └─ & ~7   → 하위 3비트 제거 → 8의 배수로 확정

두 단계를 합치면 → size를 8의 배수로 올림(ceiling)
```
