# 전역 변수 / 함수 선언 전체 분석

---

## 전체 코드

```c
static char *heap_listp; // 힙 순회의 시작점. mm_init()에서 프롤로그 풋터를 가리키도록 초기화
static char *rover;      // next-fit 탐색용: 마지막 탐색 위치 기억

static void *extend_heap(size_t words); // 힙이 부족할 때 OS에게 메모리를 더 요청
static void *coalesce(void *bp);        // 인접한 가용 블록을 병합
static void *find_fit(size_t asize);    // 가용 리스트에서 asize 이상인 블록을 탐색
static void  place(void *bp, size_t asize); // 찾은 블록에 할당 표시 + 필요 시 분할
```

---

## `static char *heap_listp`

### 역할

힙 순회의 시작점입니다. `find_fit`이 가용 블록을 탐색할 때 이 포인터부터 출발합니다.

### 초기화

```c
// mm_init 에서
heap_listp += 2 * WSIZE;  // 프롤로그 풋터 위치를 가리키도록 이동
```

```
mm_init() 이후 힙 상태:

 +0       +4            +8            +12
┌────────┬─────────────┬─────────────┬─────────────┐
│ 패딩   │  프롤로그H  │  프롤로그F  │  에필로그H  │
│  0     │  PACK(8,1)  │  PACK(8,1)  │  PACK(0,1)  │
└────────┴─────────────┴─────────────┴─────────────┘
                              ↑
                         heap_listp
                    (프롤로그 풋터 위치)
```

### 왜 `char *`인가?

`char *`는 1바이트 단위 포인터 산술을 수행합니다.
`int *`이면 `+1`이 4바이트 이동이라 `bp - WSIZE` 같은 계산이 틀어집니다.

```
char *bp = 0x1010:
  bp - 4 = 0x100C  ← HDRP 계산, 정확히 4바이트 뒤  ✓

int *bp = 0x1010:
  bp - 4 = 0x1000  ← 4 × 4 = 16바이트 뒤로 이동  ✗
```

### 생략 가능한가?

불가합니다. `find_fit`, `coalesce`, `extend_heap` 모두 이 포인터를 기준으로 힙을 순회합니다.

## `static char *rover`

### 역할

next-fit 탐색 시 마지막으로 찾은 위치를 기억합니다.
다음 탐색은 `heap_listp`가 아니라 `rover`부터 시작합니다.

### 초기화

```c
// mm_init 에서
rover = heap_listp;  // 처음엔 heap_listp와 동일하게 시작
```

### 테스트케이스 — first-fit vs next-fit

```
힙 상태:
┌──────┬───────────┬───────────┬───────────┬──────┐
│프롤H │  블록A    │  블록B    │  블록C    │에필H │
│      │ alloc=1   │ free/32B  │ free/64B  │      │
└──────┴───────────┴───────────┴───────────┴──────┘

first-fit으로 24B 3번 할당:
  1회: heap_listp ──→ 블록A(skip) ──→ 블록B 찾음
  2회: heap_listp ──→ 블록A(skip) ──→ 블록B 찾음  ← 블록A 반복 탐색 낭비
  3회: heap_listp ──→ 블록A(skip) ──→ 블록B 찾음  ← 블록A 반복 탐색 낭비

next-fit으로 24B 3번 할당:
  1회: heap_listp ──→ 블록A(skip) ──→ 블록B 찾음  (rover = 블록B)
  2회: rover(블록B) ──→ 바로 찾음                 ← 블록A 건너뜀
  3회: rover(블록B) ──→ 바로 찾음                 ← 블록A 건너뜀
```

### 생략 가능한가?

first-fit을 사용한다면 삭제해도 됩니다. next-fit을 사용할 때만 필요합니다.

## `static void *extend_heap(size_t words)`

### 역할

힙이 부족할 때 OS에게 추가 메모리를 요청합니다.
내부적으로 `mem_sbrk()`를 호출해 힙 브레이크 포인터를 늘립니다.

### 호출되는 경우

```
mm_init()   → 최초 가용 블록 생성
mm_malloc() → find_fit() 실패 시 힙 확장
```

### 테스트케이스

```
요청: mm_malloc(5000)
  asize     = ALIGN(5000 + 8) = 5008B
  CHUNKSIZE = 4096B

  extendsize = MAX(5008, 4096) = 5008B
  extend_heap(5008 / 4 = 1252 words) 호출
  1252 % 2 == 0 → 1252 × 4 = 5008B 확장

확장 전:
┌──────┬──────┬── ... ──┬──────────┬──────┐
│ 패딩 │프롤H │  블록들 │  블록N   │에필H │
└──────┴──────┴── ... ──┴──────────┴──────┘
                                           ↑ brk

확장 후:
┌──────┬──────┬── ... ──┬──────────┬──────────────────┬──────┐
│ 패딩 │프롤H │  블록들 │  블록N   │  새 가용 블록    │에필H │
│      │      │         │          │    5008B free    │      │
└──────┴──────┴── ... ──┴──────────┴──────────────────┴──────┘
                                    ↑ bp              ↑ 새 brk
```

### 생략 가능한가?

불가합니다. 힙은 한 번에 전부 확보되지 않고 필요할 때마다 늘어납니다.
`extend_heap` 없이는 최초 `CHUNKSIZE(4KB)` 이상 할당이 불가합니다.

## `static void *coalesce(void *bp)`

### 역할

인접한 가용 블록을 병합합니다.
`mm_free`와 `extend_heap` 모두 반납/확장 직후 즉시 호출합니다.

### 테스트케이스 — 병합 없을 때 vs 있을 때

```
coalesce 없을 때 (외부 단편화 누적):

┌────────┬────────┬────────┬────────┬────────┐
│ 16B    │  8B    │ 16B    │  8B    │ 16B    │
│ free   │ alloc  │ free   │ alloc  │ free   │
└────────┴────────┴────────┴────────┴────────┘

  find_fit(32B) → 실패 → extend_heap 호출
  실제로는 48B의 가용 공간이 있는데도 찾지 못함!


coalesce 있을 때:
  mm_free(두 번째 8B 블록) 호출
  → 앞 16B + 8B + 뒤 16B = 40B 병합

┌────────────────────────┬────────┬────────┐
│       40B free         │  8B    │ 16B    │
│                        │ alloc  │ free   │
└────────────────────────┴────────┴────────┘

  find_fit(32B) → 성공  ← extend_heap 불필요
```

### 4가지 케이스

```
Case 1: 앞 alloc, 뒤 alloc → 병합 없음
┌──────────┬──────────┬──────────┐
│ 이전블록  │  현재bp  │  다음블록 │
│ alloc=1  │ free     │ alloc=1  │
└──────────┴──────────┴──────────┘
                ↑ bp 그대로 반환


Case 2: 앞 alloc, 뒤 free → 뒤와 병합
┌──────────┬──────────┬──────────┐
│ 이전블록  │  현재bp  │  다음블록 │
│ alloc=1  │ 32B free │ 16B free │
└──────────┴──────────┴──────────┘
→ 병합 후:
┌──────────┬──────────────────────┐
│ 이전블록  │    48B free          │
│ alloc=1  │                      │
└──────────┴──────────────────────┘
                ↑ bp 반환


Case 3: 앞 free, 뒤 alloc → 앞과 병합
┌──────────┬──────────┬──────────┐
│ 이전블록  │  현재bp  │  다음블록 │
│ 16B free │ 32B free │ alloc=1  │
└──────────┴──────────┴──────────┘
→ 병합 후:
┌──────────────────────┬──────────┐
│       48B free       │  다음블록 │
│                      │ alloc=1  │
└──────────────────────┴──────────┘
  ↑ bp = PREV_BLKP으로 이동


Case 4: 앞 free, 뒤 free → 앞뒤 모두 병합
┌──────────┬──────────┬──────────┐
│ 이전블록  │  현재bp  │  다음블록 │
│ 16B free │ 32B free │ 16B free │
└──────────┴──────────┴──────────┘
→ 병합 후:
┌──────────────────────────────────┐
│           64B free               │
└──────────────────────────────────┘
  ↑ bp = PREV_BLKP으로 이동
```

### 생략 가능한가?

불가합니다. `coalesce` 없이는 `free`를 반복할수록 작은 조각들이 쌓여
힙이 빠르게 고갈됩니다.

## `static void *find_fit(size_t asize)`

### 역할

가용 리스트에서 `asize` 이상인 블록을 탐색합니다.
`mm_malloc`에서 `extend_heap` 전에 먼저 호출됩니다.

### 탐색 조건

```
두 조건을 모두 만족해야 선택:
  ① alloc=0  (가용 블록이어야 함)
  ② size ≥ asize  (요청 크기 이상이어야 함)
```

### 테스트케이스 — asize = 24B

```
힙 상태:
┌──────┬──────────┬──────────┬──────────┬──────┐
│프롤H │  블록A   │  블록B   │  블록C   │에필H │
│      │ alloc=1  │ free/16B │ free/64B │size=0│
│      │  32B     │          │          │      │
└──────┴──────────┴──────────┴──────────┴──────┘
  ↑
heap_listp

탐색 순서:
  블록A: alloc=1     → skip  (할당됨)
  블록B: 16 < 24     → skip  (크기 부족)
  블록C: 64 ≥ 24     → 반환  ✓

실패 시:
  에필H: size=0 → 루프 종료 → return NULL
  → mm_malloc이 extend_heap 호출
```

### 생략 가능한가?

불가합니다. 가용 블록 탐색 없이는 할당 위치를 알 수 없습니다.

---

## `static void place(void *bp, size_t asize)`

### 역할

찾은 블록에 할당 표시를 하고, 남는 공간이 충분하면 분할합니다.

### 분할 기준

```
남는 공간 = csize - asize

  ≥ 16B (2×DSIZE) → 분할  (새 가용 블록 만들 수 있음)
  <  16B           → 통째로 할당  (조각이 너무 작음)
```

### 테스트케이스

```
Case 1: csize=64B, asize=24B → 64-24=40 ≥ 16 → 분할

Before:
┌──────────────────────────────────┐
│         가용 블록 (64B)          │
└──────────────────────────────────┘

After:
┌──────────────┬───────────────────┐
│  할당 블록   │    가용 블록      │
│    24B       │      40B          │
│  alloc=1     │    alloc=0        │
└──────────────┴───────────────────┘
↑ bp 반환


Case 2: csize=24B, asize=24B → 0 < 16 → 통째로

Before:
┌──────────────────┐
│   가용 블록(24B) │
└──────────────────┘

After:
┌──────────────────┐
│   할당 블록(24B) │
│    alloc=1       │
└──────────────────┘
↑ bp 반환


Case 3: csize=28B, asize=24B → 4 < 16 → 통째로 (내부 단편화 4B)

Before:
┌──────────────────────┐
│    가용 블록(28B)    │
└──────────────────────┘

After:
┌──────────────────────┐
│    할당 블록(28B)    │  ← 4B 내부 단편화 발생
│       alloc=1        │
└──────────────────────┘
↑ bp 반환
```

### 생략 가능한가?

`mm_malloc` 안에 직접 작성할 수 있지만, 그러면 함수가 길어지고 구조가 복잡해집니다.
`find_fit`으로 위치를 찾고 `place`로 배치하는 역할 분리가 유지보수에 유리합니다.

---

## `static`을 붙이는 이유

```
외부에 노출되는 인터페이스:
  mm_init(), mm_malloc(), mm_free(), mm_realloc()  ← 사용자가 호출

내부 헬퍼 함수 (static으로 숨김):
  extend_heap(), coalesce(), find_fit(), place()   ← 외부 호출 불가
```

`static`을 붙이면 해당 `.c` 파일 안에서만 사용 가능합니다.
외부에서 직접 호출되지 않도록 캡슐화하는 역할입니다.

---

## 함수 호출 관계

```
mm_init()
  └─ extend_heap()        ← 첫 가용 블록 생성
       └─ coalesce()

mm_malloc()
  ├─ find_fit()           ← 가용 블록 탐색
  │    └─ NULL 반환 시
  ├─ extend_heap()        ← 힙 확장
  │    └─ coalesce()
  └─ place()              ← 블록 배치 및 분할

mm_free()
  └─ coalesce()           ← 인접 블록 병합
```

## 요약

| 선언 | 생략 가능? | 이유 |
|------|-----------|------|
| `heap_listp` | 불가 | 힙 순회 기준점, 모든 함수가 사용 |
| `rover` | first-fit이면 가능 | next-fit 전용, 탐색 성능 개선용 |
| `extend_heap` | 불가 | 힙 확장 없이는 고정 크기 이상 할당 불가 |
| `coalesce` | 불가 | 없으면 외부 단편화로 힙 빠르게 고갈 |
| `find_fit` | 불가 | 가용 블록 탐색 없이 할당 위치 모름 |
| `place` | 통합 가능 | `mm_malloc`에 합칠 수 있으나 구조 복잡 |
