# `mm_init()` — 힙 메모리 초기화 함수 설명

## 함수 코드

```c
int mm_init(void) {
    if ((heap_listp = mem_sbrk(4*WSIZE)) == (void*)-1)
        return -1;

    PUT(heap_listp,           0);               // 정렬 패딩
    PUT(heap_listp + WSIZE,   PACK(DSIZE, 1));  // 프롤로그 헤더
    PUT(heap_listp + 2*WSIZE, PACK(DSIZE, 1));  // 프롤로그 풋터
    PUT(heap_listp + 3*WSIZE, PACK(0, 1));      // 에필로그 헤더

    heap_listp += 2 * WSIZE;  // 프롤로그 블록 안을 가리키도록

    if (extend_heap(CHUNKSIZE/WSIZE) == NULL)   // 첫 가용 블록 생성
        return -1;
    return 0;
}
```

---

## 역할 요약

`mm_init()`은 `malloc` 라이브러리가 사용할 힙을 초기 상태로 세팅하는 함수입니다.
핵심 역할은 딱 두 가지입니다.

1. **경계 구조물 만들기** — 프롤로그 블록 + 에필로그 헤더
2. **첫 가용 공간 확보** — `extend_heap()`으로 실제 할당에 쓸 공간 생성

---

## 힙 초기 레이아웃 (낮은 주소 → 높은 주소)

```
┌──────────┬──────────────────┬──────────────────┬──────────────────┬────────────────────────────┬─────────────────┐
│  패딩    │  프롤로그 헤더   │  프롤로그 풋터   │  에필로그 헤더  │      첫 가용 블록          │  새 에필로그    │
│  4B / 0  │  PACK(8, 1) = 9  │  PACK(8, 1) = 9  │  PACK(0, 1) = 1 │  헤더 + 페이로드 + 풋터   │  PACK(0, 1) = 1 │
│  +0      │  +1W             │  +2W ← heap_listp│  +3W            │  ~4096B (CHUNKSIZE)        │                 │
└──────────┴──────────────────┴──────────────────┴──────────────────┴────────────────────────────┴─────────────────┘
                                        ↑
                               heap_listp 최종 위치
```

> `W = WSIZE = 4바이트`

---

## 단계별 실행 과정

### Step 1 — `mem_sbrk(4*WSIZE)` : 힙 공간 16바이트 확보

```c
if ((heap_listp = mem_sbrk(4*WSIZE)) == (void*)-1)
    return -1;
```

- `mem_sbrk()`는 OS에게 힙을 `4 × 4B = 16B` 늘려달라고 요청합니다.
- 실패하면(`-1` 반환) 즉시 `-1`을 반환하고 종료합니다.
- 성공하면 `heap_listp`가 새로 확보된 영역의 시작 주소를 가리킵니다.

---

### Step 2 — `PUT` 4회 : 더미 블록 기록

| 오프셋 | 기록 값 | 의미 |
|--------|---------|------|
| `+0` | `0` | 정렬 패딩 (8바이트 정렬을 위한 빈 워드) |
| `+1W` | `PACK(8, 1)` | 프롤로그 헤더 — 크기 8B, 할당됨 |
| `+2W` | `PACK(8, 1)` | 프롤로그 풋터 — 헤더와 동일 |
| `+3W` | `PACK(0, 1)` | 에필로그 헤더 — 크기 0, 할당됨 (힙 끝 표시) |

#### `PACK(size, alloc)` 매크로

```c
#define PACK(size, alloc)  ((size) | (alloc))
```

- 상위 29비트: 블록 크기
- 하위 3비트: 할당 여부 (`1` = 할당됨, `0` = 가용)
- 예: `PACK(8, 1)` = `0b...001000 | 1` = `9`

---

### Step 3 — `heap_listp += 2*WSIZE` : 포인터 이동

```c
heap_listp += 2 * WSIZE;
```

- `heap_listp`를 **프롤로그 풋터**를 가리키도록 이동합니다.
- 이후 블록 탐색 로직이 `heap_listp`를 기준으로 `±WSIZE`씩 이동하면서 블록을 찾습니다.
- 프롤로그 블록 안을 가리켜야 경계 조건 처리가 안전하게 동작합니다.

---

### Step 4 — `extend_heap(CHUNKSIZE/WSIZE)` : 첫 가용 블록 생성

```c
if (extend_heap(CHUNKSIZE/WSIZE) == NULL)
    return -1;
```

- `CHUNKSIZE`는 보통 `4096B (4KB)`입니다.
- 4개의 더미 워드(16B)만으로는 실제 `malloc` 요청을 처리할 공간이 없으므로, 초기화 시점에 바로 큰 가용 블록을 만들어 둡니다.
- 이 블록이 이후 `mm_malloc` 호출 시 `find_fit()`의 **첫 번째 후보**가 됩니다.
- 실패하면 `-1` 반환, 성공하면 `0` 반환.

---

## 프롤로그/에필로그가 필요한 이유

`coalesce()`(인접 가용 블록 병합 함수)가 블록을 합칠 때, 힙의 앞뒤 경계를 벗어나지 않도록 **절대 해제되지 않는 더미 블록**이 필요합니다.

| 블록 | 역할 | alloc 비트 |
|------|------|-----------|
| 프롤로그 헤더 + 풋터 | 힙의 **시작 경계** — `coalesce`가 왼쪽으로 더 이상 탐색하지 못하도록 막음 | `1` (항상 할당됨) |
| 에필로그 헤더 | 힙의 **끝 경계** — `size=0`이므로 "더 이상 블록 없음"을 알림 | `1` (항상 할당됨) |

> 프롤로그와 에필로그는 실제 데이터 저장에는 사용되지 않으며, 경계 조건 처리를 위한 **센티넬(sentinel)** 역할을 합니다.

## 요약

```
mm_init()
  ├─ 1. mem_sbrk(16B)        → 힙 공간 확보
  ├─ 2. PUT × 4              → 패딩 + 프롤로그 + 에필로그 기록
  ├─ 3. heap_listp += 8      → 포인터를 프롤로그 풋터로 이동
  └─ 4. extend_heap(4096B)   → 첫 가용 블록 생성 → malloc 준비 완료
```
