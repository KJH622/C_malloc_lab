# `GET` / `PUT` — 힙 읽기/쓰기 매크로

---

## 매크로 정의

```c
#define GET(p)        (*(unsigned int *)(p))
#define PUT(p, val)   (*(unsigned int *)(p) = (val))
```

---

## 역할 요약

`GET`과 `PUT`은 헤더/풋터에 저장된 4바이트 값을 읽고 쓰는 모든 블록 조작의 기반입니다.
핵심 역할은 딱 두 가지입니다.

1. **4바이트 단위 접근** — `char *`로는 1바이트만 읽히므로 `unsigned int *`로 캐스팅
2. **부호 없는 읽기/쓰기** — 비트 연산 시 부호 확장 방지

---

## `GET` — `unsigned int *`로 캐스팅하는 이유

```c
#define GET(p) (*(unsigned int *)(p))
```

### 이유 1 — 4바이트 단위로 읽기 위해

`p`는 `void *` 또는 `char *`입니다. 그냥 역참조하면 1바이트만 읽힙니다.

```
헤더 메모리 (4바이트)
 +0    +1    +2    +3
┌─────┬─────┬─────┬─────┐
│ 09  │ 00  │ 00  │ 00  │  ← PACK(8, 1) = 0x00000009
└─────┴─────┴─────┴─────┘

char *p 로 읽으면:
  *p = 0x09  (1바이트만 읽힘 → 나머지 3바이트 손실)

unsigned int *p 로 읽으면:
  *p = 0x00000009  (4바이트 전체 읽힘 → 정확)
```

헤더/풋터는 `PACK(size, alloc)`으로 만든 4바이트 값입니다.
반드시 4바이트 단위로 읽어야 크기와 alloc 비트를 정확히 가져올 수 있습니다.

### 이유 2 — `unsigned`로 읽는 이유 (부호 확장 방지)

`int *`가 아니라 `unsigned int *`로 캐스팅하는 이유는 **비트 연산 안전성** 때문입니다.

```c
// signed int의 경우 (위험)
int val = 0xFFFFFF09;       // 음수로 해석됨 (-247)
val >> 1;                   // 산술 시프트 → 부호 비트 복제 → 값 오염
val & ~0x7;                 // 의도치 않은 결과 가능

// unsigned int의 경우 (안전)
unsigned int val = 0xFFFFFF09;  // 양수로 해석됨 (4294967049)
val >> 1;                       // 논리 시프트 → 0으로 채워짐 → 안전
val & ~0x7;                     // GET_SIZE도 항상 양수로 정확히 추출됨
```

블록 크기는 절대 음수가 될 수 없으므로 `unsigned`로 읽는 것이 올바릅니다.

---

## `PUT` — 이렇게 생긴 이유

```c
#define PUT(p, val) (*(unsigned int *)(p) = (val))
```

`GET`의 역방향입니다.
읽을 때 `unsigned int *`로 캐스팅했으므로, 쓸 때도 동일하게 캐스팅해서 **4바이트를 통째로 씁니다**.

```
헤더 메모리 (4바이트)
 +0    +1    +2    +3
┌─────┬─────┬─────┬─────┐
│  ?  │  ?  │  ?  │  ?  │  ← 쓰기 전

char *p 로 쓰면:
  *p = 0x09 → 1바이트만 씀
┌─────┬─────┬─────┬─────┐
│ 09  │  ?  │  ?  │  ?  │  ← 나머지 3바이트 오염!

unsigned int *p 로 쓰면:
  *p = 0x00000009 → 4바이트 통째로 씀
┌─────┬─────┬─────┬─────┐
│ 09  │ 00  │ 00  │ 00  │  ← 정확하게 기록됨
```

### 실제 사용 예

```c
PUT(HDRP(bp), PACK(size, 1));

// 풀어쓰면:
unsigned int *header = (unsigned int *)((char *)bp - WSIZE);
*header = (size | 1);
// → 헤더에 4바이트 값이 정확히 기록됨
```

---

## GET / PUT 비교

| 항목 | GET | PUT |
|------|-----|-----|
| 목적 | 헤더/풋터 값 읽기 | 헤더/풋터 값 쓰기 |
| 캐스팅 이유 | `char *` 그대로 쓰면 1바이트만 읽힘 | `char *` 그대로 쓰면 1바이트만 써짐 |
| unsigned 이유 | 비트 연산 시 부호 확장 방지 | 같은 타입으로 일관성 유지 |
| 읽기/쓰기 단위 | 4바이트 (= WSIZE) | 4바이트 (= WSIZE) |

---

## 요약

```
GET(p)
  ├─ (unsigned int *)(p)    → char * → unsigned int * 캐스팅 (4바이트 단위 접근)
  └─ *(unsigned int *)(p)   → 역참조로 4바이트 읽기

PUT(p, val)
  ├─ (unsigned int *)(p)    → char * → unsigned int * 캐스팅 (4바이트 단위 접근)
  └─ *(unsigned int *)(p) = val  → 역참조로 4바이트 쓰기
```
