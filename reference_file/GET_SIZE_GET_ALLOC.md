# `GET_SIZE` / `GET_ALLOC` — 블록 정보 추출 매크로

---

## 매크로 정의

```c
#define GET_SIZE(p)  (GET(p) & ~0x7)
#define GET_ALLOC(p) (GET(p) & 0x1)
```

---

## 역할 요약

`GET_SIZE`와 `GET_ALLOC`은 헤더/풋터에 저장된 4바이트 값에서
크기와 할당 여부를 각각 꺼내는 매크로입니다.
핵심 역할은 딱 두 가지입니다.

1. **크기 추출** — `GET_SIZE` : 하위 3비트를 제거해 크기만 남김
2. **할당 여부 추출** — `GET_ALLOC` : 최하위 1비트만 남겨 alloc 확인

---

## 헤더/풋터 구조 — `PACK`으로 만든 값

두 매크로를 이해하려면 먼저 헤더/풋터에 값이 어떻게 저장되는지 알아야 합니다.

```c
#define PACK(size, alloc) ((size) | (alloc))
```

`size`와 `alloc`을 OR 연산으로 합쳐 4바이트 하나에 저장합니다.

```
size = 24 = 0b...0001 1000
alloc = 1 = 0b...0000 0001

PACK(24, 1) = 24 | 1
            = 0b 0000 0000 0000 0000 0000 0000 0001 1000
            | 0b 0000 0000 0000 0000 0000 0000 0000 0001
            = 0b 0000 0000 0000 0000 0000 0000 0001 1001
            = 25

┌────────────────────────────────┬───┬───┬───┐
│       크기 필드 (29비트)        │ x │ x │ a │
│              24                │ 0 │ 0 │ 1 │
└────────────────────────────────┴───┴───┴───┘
 비트 31                          2   1   0
                                          ↑ alloc 비트
```

`size`는 항상 8의 배수이므로 하위 3비트는 항상 `000`입니다.
그 빈 자리에 `alloc(1비트)`를 끼워 넣는 구조입니다.

---

## `GET_SIZE` — 크기만 꺼내기

```c
#define GET_SIZE(p) (GET(p) & ~0x7)
```

`~0x7`은 하위 3비트를 0으로 만드는 마스크입니다.

```
     0x7 = 0000 0000 0000 0000 0000 0000 0000 0111
    ~0x7 = 1111 1111 1111 1111 1111 1111 1111 1000
                                              ↑↑↑
                                         하위 3비트가 0
```

헤더 값 `PACK(24, 1) = 25`에서 크기를 추출하면:

```
  25      = 0b 0000 0000 0000 0000 0000 0000 0001 1001
& ~0x7    = 0b 1111 1111 1111 1111 1111 1111 1111 1000
           ──────────────────────────────────────────
            0b 0000 0000 0000 0000 0000 0000 0001 1000
          = 24  ✓
```

하위 3비트(alloc + 예약 비트)를 지워서 크기만 남깁니다.

---

## `GET_ALLOC` — 할당 여부만 꺼내기

```c
#define GET_ALLOC(p) (GET(p) & 0x1)
```

`0x1`은 최하위 1비트만 남기는 마스크입니다.

```
헤더 값: PACK(24, 1) = 25  → alloc=1 (할당됨)

  25    = 0b 0000 0000 0000 0000 0000 0000 0001 1001
& 0x1   = 0b 0000 0000 0000 0000 0000 0000 0000 0001
         ──────────────────────────────────────────
          0b 0000 0000 0000 0000 0000 0000 0000 0001
        = 1  ✓  (할당됨)


헤더 값: PACK(24, 0) = 24  → alloc=0 (가용)

  24    = 0b 0000 0000 0000 0000 0000 0000 0001 1000
& 0x1   = 0b 0000 0000 0000 0000 0000 0000 0000 0001
         ──────────────────────────────────────────
          0b 0000 0000 0000 0000 0000 0000 0000 0000
        = 0  ✓  (가용)
```

---

## 두 매크로를 나란히 보면

```
헤더 값: PACK(24, 1) = 25

비트 구조:
┌────────────────────────────────┬───┬───┬───┐
│       크기 (29비트) = 24       │ 0 │ 0 │ 1 │
└────────────────────────────────┴───┴───┴───┘
                                          ↑
                                     alloc 비트

GET_SIZE:  & ~0x7 = & 1111...1000  →  하위 3비트 제거  →  24
GET_ALLOC: & 0x1  = & 0000...0001  →  최하위 1비트만   →   1
```

---

## 다양한 케이스로 확인

| 헤더 값 | `PACK` | `GET_SIZE` | `GET_ALLOC` | 의미 |
|---------|--------|------------|-------------|------|
| 25 | `PACK(24, 1)` | 24 | 1 | 24B, 할당됨 |
| 24 | `PACK(24, 0)` | 24 | 0 | 24B, 가용 |
| 17 | `PACK(16, 1)` | 16 | 1 | 16B, 할당됨 |
| 16 | `PACK(16, 0)` | 16 | 0 | 16B, 가용 |
| 1 | `PACK(0, 1)` | 0 | 1 | 에필로그 헤더 |

## 요약

```
PACK(size, alloc) = size | alloc
  → 크기(상위 29비트)와 alloc(하위 1비트)을 하나의 4바이트에 저장

GET_SIZE(p)  = GET(p) & ~0x7
  → & 1111...1000  하위 3비트 제거  →  크기만 추출

GET_ALLOC(p) = GET(p) & 0x1
  → & 0000...0001  최하위 1비트만   →  alloc만 추출
```
