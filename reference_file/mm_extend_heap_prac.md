# `extend_heap()` — 힙 확장 함수

---

## 함수 코드

```c
static void *extend_heap(size_t words) {
    char *bp;
    size_t size;

    // ① 더블워드 정렬: 홀수 words면 +1 해서 짝수로
    size = (words % 2) ? (words+1) * WSIZE : words * WSIZE;

    if ((long)(bp = mem_sbrk(size)) == -1)      // ② OS에게 메모리 요청
        return NULL;

    PUT(HDRP(bp),            PACK(size, 0));     // ③ 새 가용 블록 헤더
    PUT(FTRP(bp),            PACK(size, 0));     // ③ 새 가용 블록 풋터
    PUT(HDRP(NEXT_BLKP(bp)), PACK(0, 1));       // ③ 새 에필로그 헤더

    return coalesce(bp);                          // ④ 인접 블록 연결
}
```

---

## 역할 요약

`extend_heap()`은 힙이 부족할 때 OS에게 메모리를 추가로 요청해 새 가용 블록을 만드는 함수입니다.
핵심 역할은 딱 두 가지입니다.

1. **힙 공간 확장** — `mem_sbrk()`로 OS에게 메모리 요청
2. **가용 블록 구조 완성** — 헤더/풋터 기록 후 `coalesce()`로 병합

## 힙 확장 레이아웃 (낮은 주소 → 높은 주소)

```
┌────────┬────────┬─────── ... ───┬──────────┬──────────────────────────┬──────────┐
│ 패딩   │ 프롤H  │   기존 블록들  │ (구)에필H│      새 가용 블록        │ 새 에필H │
│  4B    │PACK8,1 │               │PACK(0,1) │ 헤더 + 페이로드 + 풋터   │PACK(0,1) │
│        │        │               │← 덮어씀  │       size bytes         │          │
└────────┴────────┴─────── ... ───┴──────────┴──────────────────────────┴──────────┘
                                             ↑                            ↑
                                          bp 위치                    새 힙 끝 경계
```

> `W = WSIZE = 4바이트`, `D = DSIZE = 8바이트`

---

## 단계별 실행 과정

### Step 1 — 크기 정렬 : 더블워드 단위 보장

```c
size = (words % 2) ? (words+1) * WSIZE : words * WSIZE;
```

힙 블록은 반드시 **8바이트(더블워드) 배수**여야 합니다.
`words`가 홀수이면 1을 더해 짝수로 만든 뒤 `WSIZE(4)`를 곱합니다.

| words | 홀/짝 | 계산 | size | 8B 정렬 |
|-------|-------|------|------|---------|
| 5 | 홀수 | (5+1) × 4 | 24B | ✓ |
| 6 | 짝수 | 6 × 4 | 24B | ✓ |
| 7 | 홀수 | (7+1) × 4 | 32B | ✓ |
| 1024 | 짝수 | 1024 × 4 | 4096B | ✓ |

---

### Step 2 — `mem_sbrk(size)` : OS에게 메모리 요청

```c
if ((long)(bp = mem_sbrk(size)) == -1)
    return NULL;
```

- `mem_sbrk()`는 힙의 끝(brk 포인터)을 `size`만큼 전진시킵니다.
- 실패하면(`-1` 반환) `NULL`을 반환하고 종료합니다.
- 성공하면 `bp`가 새로 확보된 영역의 시작 주소(= 기존 에필로그 헤더 위치)를 가리킵니다.

```
확장 전
┌─────── ... ───┬──────────┐
│   기존 블록들  │  에필H   │
└─────── ... ───┴──────────┘
                            ↑ brk (힙 경계)

        mem_sbrk(size) 호출 → brk를 size만큼 전진
                  ↓

확장 후
┌─────── ... ───┬──────────┬─────────────────────┐
│   기존 블록들  │  에필H   │   size bytes (날것)  │
└─────── ... ───┴──────────┴─────────────────────┘
                            ↑ bp               ↑ 새 brk
                    (아직 구조 없음)
```

---

### Step 3 — `PUT` 3회 : 가용 블록 구조 기록

```c
PUT(HDRP(bp),            PACK(size, 0));  // 가용 블록 헤더
PUT(FTRP(bp),            PACK(size, 0));  // 가용 블록 풋터
PUT(HDRP(NEXT_BLKP(bp)), PACK(0, 1));    // 새 에필로그 헤더
```

#### 매크로 위치 계산

```
  HDRP(bp)          = bp - WSIZE          (헤더: 페이로드 바로 앞 4B)
  FTRP(bp)          = bp + size - DSIZE   (풋터: 블록 끝 -8B)
  NEXT_BLKP(bp)     = bp + size           (다음 블록 페이로드)
  HDRP(NEXT_BLKP)   = bp + size - WSIZE  (새 에필로그 위치)
```

```
기록 전 (날것의 메모리)

 bp-4W      bp                         bp+size-8  bp+size-4
  ┌──────────┬──────────────────────────┬──────────┬──────────┐
  │ 구 에필H │       (쓰레기 값)        │    ??    │    ??    │
  └──────────┴──────────────────────────┴──────────┴──────────┘
  HDRP(bp)                              FTRP(bp)  HDRP(NEXT)


기록 후 (구조 완성)

 bp-4        bp                         bp+size-8  bp+size-4
  ┌──────────┬──────────────────────────┬──────────┬──────────┐
  │PACK(sz,0)│     (페이로드 공간)      │PACK(sz,0)│PACK(0,1) │
  │ 가용헤더 │          free            │ 가용풋터 │새 에필로그│
  └──────────┴──────────────────────────┴──────────┴──────────┘
  alloc=0                               alloc=0    alloc=1
```

> **기존 에필로그가 헤더로 재활용되는 이유**
> `bp`는 기존 에필로그 헤더 자리를 가리킵니다.
> `HDRP(bp) = bp - WSIZE`이므로 기존 에필로그 위치에 새 가용 블록 헤더를 덮어씁니다.
> 에필로그는 항상 새 블록의 끝(`bp + size - WSIZE`)으로 이동합니다.

---

### Step 4 — `coalesce(bp)` : 인접 블록 병합

```c
return coalesce(bp);
```

- 새 가용 블록 **바로 앞 블록**이 이미 free 상태라면 하나로 합칩니다.
- 4가지 경우를 처리합니다.

```
Case 1 — 앞 alloc, 뒤 alloc → 병합 없음
┌──────────┬──────────────┬──────────┐
│  이전블록 │  새 가용블록 │  에필H   │
│ alloc=1  │   alloc=0    │ alloc=1  │
└──────────┴──────────────┴──────────┘
                ↑ bp 그대로 반환


Case 3 — 앞 free, 뒤 alloc → 앞과 병합
┌──────────┬──────────────┬──────────┐
│  이전블록 │  새 가용블록 │  에필H   │
│ alloc=0  │   alloc=0    │ alloc=1  │
└──────────┴──────────────┴──────────┘
      ↓ 병합
┌────────────────────────┬──────────┐
│     합쳐진 가용 블록    │  에필H   │
│  크기 = 이전 + 새로운  │          │
└────────────────────────┴──────────┘
↑ 이전 블록의 bp 반환
```

> `extend_heap`에서 뒤(에필로그)는 항상 `alloc=1`이므로
> 실제로는 **Case 1** 또는 **Case 3**만 발생합니다.

---

## `PACK(size, alloc)` 매크로

```c
#define PACK(size, alloc)  ((size) | (alloc))
```

- 상위 29비트: 블록 크기
- 하위 3비트: 할당 여부 (`0` = 가용, `1` = 할당됨)
- 예: `PACK(4096, 0)` = `0b...1000000000000 | 0` = `4096`

---

## 요약

```
extend_heap(words)
  ├─ 1. 크기 정렬          → 홀수 words면 +1, 8B 배수 보장
  ├─ 2. mem_sbrk(size)    → OS에게 size 바이트 요청, bp 획득
  ├─ 3. PUT × 3           → 가용 헤더 + 가용 풋터 + 새 에필로그 기록
  └─ 4. coalesce(bp)      → 이전 블록 free면 병합 후 bp 반환
```
